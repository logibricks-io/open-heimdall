## 时代的断层

每六周，前沿都在移动。

一个新模型到来——比之前的更快、更便宜、更聪慧。未来悄然改写自己。而在世界的某个角落，上万个基于上个季度模型构建的应用，开始它们缓慢的过时之旅。

这是 AI 时代的核心张力。**智能的基底，正以人类前所未有的速度演化**——而构建于其上的应用，本质上渴望稳定。模型不允许它们稳定。

Heimdall 就立于这道断层之上。

## 我们的论点

> *应用不应当知道它运行在哪个模型之上。*

它只需要知道自己需要什么——推理、视觉、对话、代码、判断——然后相信桥头的守门人会如期提供。

模型层应以自己的节奏演化，应用层应以自己的节奏演化。两者之间存在一层薄膜——一位通晓两种语言、为两个世界服务的守护者。

当下一个前沿模型发布的那一刻，你的应用并不会知晓。它只会变得更好。

## 我们正在构建什么

一道桥关。通向一切已存在的模型，与一切将存在的模型。

在一个 OpenAI 兼容的端点之后，住着六件器物，每一件以阿斯加德的圣物命名，每一件回答着 AI 技术栈的一个古老问题：

> **Bifröst** 彩虹桥 ——*这个请求应当奔向何方？*  
> **Aegis** 神盾 ——*我应当如何庇护我所服务的一切？*  
> **Sleipnir** 八足神驹 ——*我应当如何永不耗尽力量？*  
> **Forge** 锻造工坊 ——*我应当如何磨砺我所发出的言辞？*  
> **Gjallarhorn** 预警号角 ——*我的领域中，正发生什么？*  
> **Compass** 罗盘 ——*这一切，代价几何？*

每一件器物的细节，可见于[文档](https://github.com/logibricks-io/open-heimdall/tree/main/docs)。或者，你也可以直接把应用指向 Heimdall——然后忘记这些问题曾经是你的问题。

## 我们决不妥协的事

**稳定胜于新潮。** 你的应用不是为了每季度被重写一次而生的，它们也不会被如此对待。

**透明胜于魔法。** 每一次路由决策、每一个 token、每一分钱——皆可观测、可归因、可回放。在你与你的账单之间，没有黑盒。

**兼容胜于发明。** 我们不要求你学习一套新的 API。我们说你早已说着的语言。

**开放胜于围墙。** 这是 *Open* Heimdall。这道桥关属于你——可读、可运行、可派生、可锻造。**如此重要的基础设施，理应属于公地。**

## 开始

```bash
# 拉取镜像并启动
docker run -d -p 8080:8080 ghcr.io/logibricks-io/open-heimdall:latest

# 或从源码构建
git clone https://github.com/logibricks-io/open-heimdall.git
cd open-heimdall && make run
```

将你的应用指向 `http://localhost:8080/v1`。

继续构建。
